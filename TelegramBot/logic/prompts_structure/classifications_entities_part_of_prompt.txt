Значения "category" и "subcategory" для type - GeoPlace:
    "category" для type - GeoPlace отсутствует!
    "subcategory" для type - GeoPlace отсутствует!

Правила для работы с этими списками при заполнении:
    1. Ты ставишь "category" и "subcategory" только из этих списков.
    2. Ставишь по логическому принципу, то есть что объекту больше подохдит.
    3. В поле "category" может быть только одно значение, в поле "subcategory" несколько
    4. Поле "subcategory" ВСЕГДА должно быть массивом:
      - Для Infrastructure: массив с одним или несколькими значениями из списка subcategory
      - Для Biological и GeoPlace: всегда пустой массив []

Примеры:
  "primary_entity": {{
    "name": "Байкальский музей",
    "type": "Infrastructure", 
    "category": "Достопримечательности",
    "subcategory": ["Музеи"]
  }}
  "primary_entity": {{
    "name": "Природоохраняемые территории",
    "type": "Infrastructure", 
    "category": "Природный объект",
    "subcategory": ["Заказники", "Заповедники"]
  }}
  "primary_entity": {{
    "name": "Лопость турбины",
    "type": "Infrastructure", 
    "category": "Достопримечательности",
    "subcategory": ["Памятники"]
  }}
  "primary_entity": {{
